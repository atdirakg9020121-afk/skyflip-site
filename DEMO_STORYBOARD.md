# デモ動画・スクショ 絵コンテ＆撮影ガイド

自宅で SkyFlip を録画・撮影するときの設計書。これに沿えば「何を何秒見せるか」で迷わない。
SkyFlip の強み（時刻で空が変わる／太陽・月が動く／本物の月齢／天気連動／カレンダー）を最短で伝えるのが目的。

---

## 撮る前の準備
- アプリを実際の天気が「晴れ」か「雨」のときに起動して撮ると映える（雨は特に画になる）
- 画面録画ツール: Windows 標準は Xbox Game Bar（Win+G）。きれいに録るなら OBS Studio（無料）推奨
- 録画解像度: 1920×1080 で録り、用途別に書き出す
- 時刻が変わる演出は実時間だと遅いので、**アプリ側にデモ用の早送り/時刻変更があれば活用**。無ければ各時間帯を個別に撮って繋ぐ

---

## ① hero.mp4（サイトのトップ背景・最重要）
- **長さ: 8〜12秒・ループ前提**（先頭と末尾が自然に繋がる絵にする）
- **音なし**（サイトでは muted 自動再生）
- 字幕・UIは入れない。純粋に「美しい動き」だけを見せる（サイトのコピーが言葉を担当する）

### カット割り（例）
| 秒 | 見せるもの |
|---|---|
| 0.0–3.0 | フリップ時計が1〜2回パタンとめくれる。背景は昼の青空、雲がゆっくり流れる |
| 3.0–6.0 | 空の色が夕方へなめらかに変化、太陽が傾いて沈んでいく |
| 6.0–9.0 | 夜空へ。月が昇り、本物の満ち欠けの月が浮かぶ |
| 9.0–12.0 | （雨バージョンを少し）雨が画面を伝い、空が暗く沈む → 先頭の昼へ自然に戻す |

> ループを自然にするコツ: 末尾を先頭と同じ「昼・同じ時計の数字」に戻すと繋ぎ目が目立たない。

## ② SNS用 GIF / 短尺動画（X・Reddit拡散用）
- **長さ: 3〜6秒**。冒頭1秒で掴む（SNSは最初の一瞬が勝負）
- いちばんインパクトのある絵を先頭に: **「雨が画面に降る」または「月の満ち欠け」**
- GIFは重くなりがち → 720p・15fps程度に圧縮。動画(mp4)が使えるならその方が軽くて高画質
- X用は正方形(1:1)か横(16:9)。最初のフレーム(サムネ)が魅力的になる瞬間を選ぶ

### 構成（例・5秒）
| 秒 | 見せるもの |
|---|---|
| 0–1.5 | 雨が画面に伝う（つかみ） |
| 1.5–3.5 | 時刻が進み空の色が変わる（早送り感） |
| 3.5–5.0 | 夜の月（満ち欠け）でしっとり締め |

## ③ Store スクリーンショット（静止画・4〜8枚）
動画と違い「一目で機能が分かる」静止画。各画像に添える説明文は `STORE_LISTING.md` にある。

| # | 撮る場面 | 対応キャプション(日) |
|---|---|---|
| 1 | 昼の晴れ＋時計がはっきり見える基本画面 | いまの天気が、そのまま動く時計に。 |
| 2 | 雨で画面に雨＋空が暗い状態 | 雨の日は、画面にも雨が伝う。 |
| 3 | 夜空＋満ち欠けの月がはっきり見える | 夜空の月は、今夜の本物と同じ。 |
| 4 | カレンダー表示中の画面 | ボタンひとつで、月のカレンダーに。 |
| 5(任意) | 有料テーマ（桜 or 銀河）の画面 | 空気を、着せ替える。 |

- 解像度: 1366×768 以上（Store要件）。横向き推奨
- 余計なデスクトップアイコン・通知が写り込まないよう、撮影前にデスクトップを片付ける

## 仕上げメモ
- hero.mp4 ができたら `assets/hero.mp4` に置くだけでサイトに反映（index.html が参照済み）
- 先頭フレームを書き出して `assets/hero-poster.jpg` にすると、動画読込前もきれいに見える
- スクショのうち1枚は OGP画像(`assets/ogp.png`)の素材にも流用できる（OGP_DESIGN.md 案A/B参照）
- ⚠️ 要推敲: 実際に撮ってみて、テンポが遅い/掴みが弱いと感じたら秒数を詰める。最初の1秒を最優先で。
